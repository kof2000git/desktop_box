# 标签盒子 + 分类扩展 设计

- 日期：2026-06-14
- 基线版本：v1.0.1
- 目标版本：v1.1

## 目标

1. **功能1 · 扩展分类**：把桌面文件整理为更明确的类别——音频、视频、快捷方式、文档（md/txt/pdf/Office 齐全）、应用程序等。
2. **功能2 · 标签盒子**：把多个分类盒子合并到同一位置，顶部一排 TAB 切换显示，省桌面空间。

---

## 功能1：扩展类别

### 最终分类集（9 类）

`应用程序 / 文档 / 图片 / 压缩包 / 视频 / 音频 / 快捷方式 / 文件夹 / 其他`

### 相对现状的变更

| 分类 | 变化 | 补充扩展名 |
|---|---|---|
| 应用程序 | 原「程序」改名 | `+ .appx .msix` |
| 音频 | 原「音乐」改名 | `+ .ape .mka .opus .aiff .mid .midi` |
| 快捷方式 | 🆕 独立成类 | — |
| 文档 | 保留（已含 md/txt/pdf/office/wps） | — |

### 快捷方式归类策略（核心）

`.lnk` → 解析目标路径：

- 目标是**文件夹** → 归「文件夹」（与真文件夹一致）
- 其它（exe/文档/网址/不可解析）一律 → 归「快捷方式」

即快捷方式不再穿透解析到目标类型，统一进「快捷方式」。

### 影响范围

- `Services/CategorizerService.cs`
- `Tests/CategorizerServiceTests.cs`（更新命名断言 + 新增用例）

仅影响"一键整理"的分类；拖放/打开逻辑不变。

## 功能2：标签盒子

### 数据模型

`Box` 新增字段 `List<BoxTab> Tabs`：

- `Tabs` 为空 → **普通模式**，使用现有 `Items`（完全向后兼容）
- `Tabs` 非空 → **标签模式**

```csharp
public class BoxTab {
    public Guid Id { get; set; } = Guid.NewGuid();
    public string Name { get; set; } = "标签";
    public List<BoxItem> Items { get; set; } = new();
}
```

旧 `boxes.json` 的 `Box` 无 `Tabs` 字段 → 反序列化为空 → 普通模式，**无破坏**。

### 交互

- 外观与普通盒子一致（半透明卡片，可拖动/缩放），标题栏下方多一行可水平滚动的 TAB。
- 点 TAB 切换当前显示，当前 TAB 高亮。
- **拖放**：拖文件进标签盒子 → 加到**当前选中** TAB。
- **TAB 右键**：重命名 / 删除（仅空 TAB 可删）。
- **TAB 行末「+」**：新建空 TAB。
- **删除盒子**：标签盒子需所有 TAB 都为空才能删（沿用"非空不可删"规则）。

### 整理流程

一键整理：分类后产出 **1 个标签盒子**（名「桌面整理」），每个分类 = 一个 TAB。
还原整理：按 `BoxId` 删除该标签盒子（等价于现在删多个分类盒子）。

### 手动合并 / 拆分（盒子右键菜单）

- 普通盒子 →「转为标签盒子」（成为含 1 个 TAB 的标签盒子）
- 标签盒子 →「合并其它盒子进来…」（对话框选择）→ 被选盒子作为新 TAB 合并、并从桌面移除
- 标签盒子 →「拆分为独立盒子」→ 每个 TAB 还原为独立盒子

### 影响文件

- `Models/Box.cs`（新增 `Tabs` + `BoxTab`）
- `ViewModels/BoxViewModel.cs`（Tabs 集合 + SelectedTab + 标签/普通模式）
- `ViewModels/MainViewModel.cs`（整理默认合并、合并/拆分命令、拖放归当前 TAB、删除标签盒子）
- `Controls/BoxControl.xaml(.cs)`（标签模式 UI：TAB 行、+、切换、右键）
- `Controls/ItemTile.xaml.cs`（`OnDelete` 改为同时遍历 `Items` 与 `Tabs[].Items`）
